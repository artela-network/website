The Chain-Native pattern allows developers to extend the capabilities of the blockchain base layer by adding custom extensions, enabling them to tailor the base layer features to fit their specific use cases. 
Application Layer
The layer for developers to build smart contracts.
Extension Layer
Developers can build system-level customized extensions upon base layer and compose dApps with extensions and smart contracts.
Blockchain Base Layer
A core basic layer with system-level components. (mempool, consensus, networking, etc)
Enhance dApp by Aspect Programming
Aspect is an extension that runs in the extension layer, providing access to all APIs provided by the base layer. What sets Aspects apart from smart contracts is their ability to manage the entire lifecycle of a transaction and interact with the base processing context. The following are just a few examples of what Aspects can do:
· Customizable transaction process.
· Construct blocks with customized rules.
· Access the execution context and create reliable asnyc tasks.
Customization & Composability in One Network
Builders have the ability to develop or expand aspects to achieve chain-native customization within a single public blockchain network. This breakthrough capability overcomes the drawbacks and limitations of smart contract dApps and app-specific chains, by providing dApps with the unprecedented ability to achieve native-scale composability and system-level customization.
